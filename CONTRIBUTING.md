# Contributing

Contributions must preserve the reviewer’s independence and non-authority
boundary.

- Use a branch-scoped pull request; do not push substantive changes to `main`.
- Keep the PR narrow and state whether it changes trust, permissions, schemas,
  integrations, or rate limits.
- Include tests or deterministic synthetic fixtures for behavior changes.
- Do not include secrets, private review input, provider credentials, or
  machine-specific paths.
- The author cannot be the independent final reviewer of the change.
